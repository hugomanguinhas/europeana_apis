Example queries

  • Openly licensed content - &reusability=open
  • Limit to items with >4 megapixel images - &qf=IMAGE_SIZE:extra_large
  • Colour search - add one or more colours e.g. &colourpalette=%23556B2F (Dark Olive Green)
  • Subject searches e.g. paintings - &qf=what:portrait

Searching for locations

"Louvain - le Théâtre et la rue de la Station". Contributed by Arietta Ruß as part of the Europeana 1914-18 project. CC BY-SA.

As many photographic related applications require location-based material, it’s worth giving some extra advice on how to find these.

  • Within the Europeana API there is the ability to search for geotagged materials using a bounding box. You can append a parameter such as
    &qf=pl_wgs84_pos_lat:[50.8628526691932+TO+50.89686185152743]+AND+pl_wgs84_pos_long:[4.644899368286132+TO+4.756135940551758]
    This has been used on the PostcodePast site that has a small script to generate a geoJson feed for display using LeafletJS - e.g. see http://postcodepast.com/data/europeana.php?bbox=4.644899368286132,50.8628526691932,4.756135940551758,50.89686185152743 (feel free to use this for testing, but for production purposes you can find the code under an open license on GitHub)
  • The presence of geotagged material can be limited if providers have not given geotags with their metadata. There are a few possible ways to overcome this, for example by searching for a location name. You can simply type the name as a search term e.g. q=Leuven or if that returns poor results try the more specific &qf=what:Leuven OR Louvain OR Löwen
  • An example of this approach has been implemented on PostcodePast which uses the Wikipedia API to find geotagged points of interest and then searches these names within Europeana (and Flickr & DPLA) - see e.g. http://postcodepast.com/#lat=50.8798438&lng=4.700517600000012&zoom=14. The concept and API calls behind this are illustrated in this Google Sheet (please make a copy for your own use before editing/changing).
